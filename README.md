# Doc Reader — приватная веб-читалка `.md`

Статический сайт, который читает Markdown-файлы из **приватного** GitHub-репозитория и показывает их в браузере. Сервер не нужен.

## Как устроено

```
┌─ Публичный репо (этот код) ──┐        ┌─ Приватный репо ──────┐
│ index.html / app.js / css    │        │ notes/*.md            │
│ → GitHub Pages (сайт)        │        │ docs/*.md  …          │
└──────────────┬───────────────┘        └───────────┬───────────┘
               │  открываешь в браузере              │
               ▼                                      │
        вводишь токен один раз ───── GitHub API ──────┘
        (хранится только в браузере)   (Contents/Blobs, CORS-ок)
```

- Сайт-оболочка публичная, но **без токена не показывает ничего**.
- Документы остаются в приватном репозитории. Токен — fine-grained, **read-only, на один репо**, лежит только в `localStorage` браузера.
- Документы тянутся через GitHub API (Contents/Blobs) — единственный путь, который работает с CORS для приватных репо прямо из браузера.
- Папки внутри репозитория становятся группами в боковой панели.
- Обновление: просто добавляешь/меняешь `.md` в приватном репозитории — приложение всегда показывает свежую версию.

## Безопасность

- Используется **fine-grained, read-only токен на один репозиторий** — минимальный ущерб, если он утечёт.
- Токен лежит в `localStorage` браузера; кнопка ⚙ позволяет его сменить. Не коммить токен никуда.
- Сам сайт на Pages **публичен по URL** (приватная видимость Pages — только в GitHub Enterprise), поэтому защита держится на том, что без токена контент не отдаётся, а репозиторий с документами приватный.
- Markdown рендерится через `marked` и санитайзится `DOMPurify` (защита от XSS в `.md`).
